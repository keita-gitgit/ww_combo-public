# 鳴潮コンボ記録ツール — 事前調査と設計方針

作成日: 2026-07-12

## 1. 鳴潮（Wuthering Waves）とはどんなゲームか

Kuro Games が開発するオープンワールドのアクションRPG。PS5 / PC / スマホでプレイ可能。
戦闘は 3 人パーティを組み、1 人を操作しながらリアルタイムでキャラを切り替えて戦うスタイル。
切り替え時に発動するスキルがあるため「どの順番で・どの技を・どのタイミングで出すか」＝ローテーション（コンボ）が攻略の核になる。

### 戦闘のアクション体系（全キャラ共通の枠組み）

| 分類 | 内容 |
| --- | --- |
| 通常攻撃 | 連打で 1段目→2段目→… と派生するコンボ。押すタイミング・状況で重撃 / 空中攻撃 / 回避反撃に変化 |
| 重撃 | 通常攻撃ボタン長押し |
| 共鳴スキル | クールタイム付きのメインスキル |
| 共鳴解放 | いわゆる必殺技。共鳴エネルギーが溜まると使用可能 |
| 協和破壊 | 敵の不協和値が満タンの時に発動できる共通アクション |
| 共鳴回路（強化状態） | キャラ固有のゲージ。溜まると「強化通常」「強化スキル」など特殊な派生技が解放される |
| 変奏スキル | 協奏エネルギーが 100 溜まった状態でキャラ交代すると、**出てくる側**が発動する攻撃 |
| 終奏スキル | 同条件で、**下がる側**が発動するバフ・攻撃 |
| 音骸（エコー）スキル | 装備した音骸による召喚攻撃・変身など |
| 回避 / ジャンプ / 落下攻撃 | 移動系アクション。コンボの繋ぎに使う |

さらにキャラごとに固有の技名・派生（例: フロロヴァの「ヘカテー」召喚、千咲のハサミ攻撃→チェンソーなど）があり、
**「共通アクション + キャラ固有アクション」の 2 層構造**になっている。これがツールのデータモデルの基本になる。

### PS5 のデフォルトボタン配置（参考）

- □: 通常攻撃 / △: 共鳴スキル / R1: 共鳴解放 / ×: ジャンプ / L1+□ 等: 音骸スキル / 十字キー: キャラ交代
- キー配置はゲーム内で自由に変更可能 → **ツール側でもボタン表記はユーザーが編集できる**ようにする
  （手書きメモでは R2 / L2+□ / R2+△ 等が使われており、カスタム配置の可能性が高い）

## 2. 手書きメモの分析（ツールが表現すべきもの）

添付ノート（パーティ: 秧秧 / フローヴァ / 千咲）から読み取れる記録の構造:

1. **パーティ定義**: 3 キャラの組み合わせ
2. **ローテーション**: 「キャラ名 : アクション列」を時系列に並べたもの。同じキャラが何度も登場する
3. **アクション列**: `通常攻撃 / 共鳴スキル / 共鳴解放 / 重撃` のようにスラッシュ区切りの短い列
4. **補足情報**が随所に付く:
   - 注意書き（※敵に当ててゲージを消費する必要あり、※時間に少し余裕あり）
   - 「自然発生」マーク（自分で入力せず勝手に出る技）
   - 入力メモ（ほぼ同時押し R2+△、○ → R2 → R2 → L2+□）
   - モーションの説明（ヘカテーを投げて倒れこむ）

→ ツールに必要なのは単なる「技の並び」ではなく、**ステップごとに注釈・入力ボタン・フラグ（自然発生/同時押し）を持てる構造**。

## 3. ツールの設計方針

### 前提条件

- 利用者は自分ひとり。配布しない
- スマホでポチポチ選択して記録・閲覧したい（PS5 でプレイしながら片手でスマホを見る想定）
- サーバーやアカウントは不要にしたい

### 推奨アーキテクチャ: 静的 Web アプリ（PWA）+ GitHub Pages

| 項目 | 選定 | 理由 |
| --- | --- | --- |
| 形態 | ブラウザで動く 1 枚の Web アプリ | アプリストア配布不要。スマホのホーム画面に追加すればアプリ同然に使える |
| ホスティング | GitHub Pages | このリポジトリから無料で公開でき、URL を開くだけで使える |
| データ保存 | localStorage（+ JSON エクスポート/インポート） | サーバー不要。機種変更やバックアップはエクスポートで対応 |
| 技術 | Vite + React + TypeScript | スマホ向け UI を部品化しやすく、型でデータモデルを守れる |

キャラクターの公式データ（一覧・技名）はコミュニティ製 API（resonance-rest/api 等）も存在するが、
外部依存が増えるうえ日本語名や「自分がコンボ表記で使う技名」（例: ハサミ1、チェンソー）とは一致しない。
**キャラ・技はツール内で自分で登録するマスタ管理方式**とし、初期データとして代表キャラを同梱する程度に留めるのが現実的。

### データモデル（案）

```
Character（キャラマスタ）
  - id, 名前, 属性, 武器種
  - actions: CharacterAction[]   … このキャラ専用の技リスト

CharacterAction（技マスタ）
  - 初期技は全キャラ共通の16項目（通常1〜5、共鳴スキル、重撃、空中攻撃、共鳴解放、協和破壊、音骸、変奏、終奏、回避、ジャンプ、落下攻撃）
  - 段数、強化状態、特殊派生は利用者が自由に追加・改名する（例: ハサミ1、チェンソー…）
  - 種別（通常/重撃/スキル/解放/音骸/変奏/終奏/移動/固有）
  - デフォルトのボタン表記（例: R2, L2+□）※編集可

Party（パーティ）
  - 名前, メンバー: Character×3

Combo（コンボ / ローテーション）
  - 対象パーティ, タイトル, 全体メモ
  - steps: ComboStep[]

ComboStep（1 行 = 手書きメモの 1 行に相当）
  - キャラ（パーティから選択）
  - アクション列: ComboAction[]
  - 行メモ（※注意書き）

ComboAction（1 つの技）
  - 技（そのキャラの actions + 共通アクションから選択）
  - フラグ: 自然発生 / ほぼ同時押し（前の技と）
  - ボタン表記の上書き, 個別メモ
```

### 画面構成（スマホ縦持ち前提）

1. **キャラ管理**: キャラの登録・編集。技リストもここで育てる
2. **パーティ作成**: 登録済みキャラから 3 人選ぶ
3. **コンボ編集**: パーティを選ぶ → キャラをタップ → 技をタップで 1 手ずつ追加。
   技ボタンは大きめにして「ポチポチ」で組めるようにする。長押し等でフラグ・メモを付与
4. **コンボ閲覧**: 手書きメモと同じ「キャラ : 技 / 技 / 技」形式で大きく表示。
   練習中に見るための画面なので文字サイズ・コントラスト重視
5. **設定**: ボタン表記の編集、JSON エクスポート/インポート

### 開発の進め方（フェーズ案）

- **Phase 1**: プロジェクト雛形 + データモデル + キャラ登録画面（localStorage 保存）
- **Phase 2**: パーティ作成 + コンボ編集（タップで技を積む UI）
- **Phase 3**: コンボ閲覧画面（手書きメモ互換の表示）+ フラグ/メモ/ボタン表記
- **Phase 4**: GitHub Pages 公開 + PWA 化（ホーム画面追加・オフライン動作）
- **Phase 5**: エクスポート/インポート、並べ替え・複製などの利便性向上

## 5. 音骸スコア機能のマスターデータ

調査日: 2026-07-24 / 対象: Ver3.5

音骸スコア計算・スクリーンショットOCR・共有カード表示で共通利用するため、
以下のデータをアプリ同梱の静的マスターとして管理する。サーバーや外部APIへは依存しない。

| データ | ファイル | 内容 |
| --- | --- | --- |
| 音骸 | `src/data/echoes.json` | 180種。安定ID、日本語名、COST、所属ハーモニー、通常/ナイトメア/響き渡る共鳴の区分 |
| ハーモニー | `src/data/sonataEffects.json` | 34種。実装Ver、必要セット数、効果の要約 |
| ステータス | `src/data/echoStats.ts` | COST別メインステータス、固定枠、サブステータス13種の全数値段階、OCR用別表記 |

### 設計上の決定

- ユーザーが保存する音骸は表示名ではなく `echo-001` のような安定IDでマスターを参照する。
- 性能が同一でスキンとして扱われる異相音骸は別個体として重複登録しない。
- ハーモニーも同様に `sonata-01` のような安定IDで参照し、名称変更の影響を受けないようにする。
- 音骸名は画面表示用の正式名とOCR照合用の別表記を分離する。
- 元スクリーンショットはlocalStorageへ保存せず、将来のOCR実装では読み取った数値だけを保存する。
- `npm run build` の最初に件数、ID重複、未登録ハーモニー参照、セット効果の欠損を検査する。
- 音骸画像は今回のマスターには含めない。共有カード実装時に、オフライン利用と権利関係を確認したうえで
  ローカル同梱素材として別管理する。

### 調査・照合元

- GameWith「音骸一覧」: 日本語名、COST、ハーモニー所属、Ver3.5追加分
- 鳴潮 Wiki*「音骸一覧」「音骸/ハーモニー効果」「音骸・データドック」:
  音骸一覧の照合、セット効果、COST別メインステータス、サブステータス数値

次の実装段階では、このマスターを参照する手入力フォームとスコア計算を先に作り、
その入力フォームをOCR結果の確認・修正画面として再利用する。
